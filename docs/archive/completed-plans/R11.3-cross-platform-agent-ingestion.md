# R11.3 — Cross-platform agent ingestion

**Parent strategy:** [../../PROGRAM-REVIEW-2026-04.md](../../PROGRAM-REVIEW-2026-04.md)
**Tier:** A
**Task ID:** `32108772-f9ba-4cd5-8440-820a44caad75`

## 1. Goal

Make Vibe Dash populate itself automatically from **any** AI coding tool —
Claude Code, Cursor, Codex, Copilot Workspace, Aider, and future entrants —
without the agent having to explicitly call MCP tools. This is the moat
against vendor lock-in: Vibe Dash is the layer **above** any one vendor's
runtime.

## 3. Data model changes

Migration in `schema.ts`:

- New table `ingestion_sources` (id TEXT PK, name TEXT, kind
  ENUM('claude_code','cursor','codex','copilot','aider','generic'),
  token_hash TEXT, project_id FK NULL, active INT DEFAULT 1, created_at
  TEXT, last_event_at TEXT NULL).
- New table `ingestion_events` (id TEXT PK, source_id FK, received_at
  TEXT, raw_payload TEXT JSON, normalized_kind
  ENUM('activity','cost','tool_call','file_change','session_start',
  'session_end'), task_id FK NULL, agent_id FK NULL, processed INT
  DEFAULT 0) — append-only buffer; downstream workers materialize into
  `activity` / `costs` / `file_locks`.
- Index `idx_ingestion_events_unprocessed` on (processed, received_at).

## 4. Server surface

New module `server/ingestion/`:
- `index.ts` — Express sub-router mounted at `/api/ingest`.
- `normalizer.ts` — dispatch on `kind`:
  - `fromClaudeCodeHook(payload)` — handles `PreToolUse`/`PostToolUse`/
    `Stop`/`UserPromptSubmit` hook shapes (well-documented).
  - `fromCursorEvent(payload)` — Cursor's agent-log JSON.
  - `fromCodexEvent(payload)` — OpenAI Codex CLI log shape.
  - `fromCopilotEvent(payload)` — GitHub Copilot Workspace event shape.
  - `fromAiderEvent(payload)` — Aider's JSON output when run with
    `--json`.
  - `fromGeneric(payload)` — schema in `docs/GENERIC-INGEST-SCHEMA.md`
    (new doc, see §9 in R13.2).
- `materializer.ts` — reads `ingestion_events` where processed=0, calls
  existing `logActivity`/`logCost`/`reportWorkingOn` helpers; sets
  processed=1. Runs on an interval + kicked on every insert.
- `auth.ts` — bearer-token check against `ingestion_sources.token_hash`.

Endpoints:
- `POST /api/ingest/:source_kind` — accepts bearer token, stores raw
  payload, enqueues. Returns 202.
- `POST /api/ingest/sources` (authed admin) — create source, returns
  generated token (shown once).
- `GET /api/ingest/sources`, `DELETE /api/ingest/sources/:id`.
- `GET /api/ingest/events?source_id=&since=` — debug / replay.
- Heartbeat `POST /api/ingest/:source_kind/heartbeat` updates
  `last_event_at`.

MCP tools:
- `list_ingestion_sources`, `create_ingestion_source`,
  `rotate_ingestion_token` (returns new token once).

WebSocket: `ingestion_event_received`, `ingestion_source_silent` (no
events for > N minutes).

## 9. Out-of-scope

- OpenTelemetry semconv export (de-prioritized in program review).
- Slack/Discord notification — continue to use webhooks.
- Agent *outputs* replayed end-to-end — just ingestion, not re-run.

## 10. Deprecation companion

Once this lands, many explicit MCP tools become redundant for the common
flow (agents no longer need to call `log_activity` if hooks cover it).
Do not remove yet — the explicit tools remain the documented "pull" path
for agents that don't have hook support. Flag in R11.4 for review.
