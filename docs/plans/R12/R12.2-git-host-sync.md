# R12.2 — Git host sync (GitHub / GitLab)

**Parent strategy:** [../../PROGRAM-REVIEW-2026-04.md](../../PROGRAM-REVIEW-2026-04.md)
**Tier:** B
**Task ID:** `19944e02-c743-4857-8e56-ff9cde9f9152`

## 1. Goal

Two-way sync between Vibe Dash tasks and GitHub/GitLab issues; surface PR
status on tasks. Removes the *"my team lives in GitHub"* objection while
keeping Vibe Dash as the source of truth for AI-driven work.

## 2. Context & prior art

Read first:
- [server/db/tasks.ts](../../../server/db/tasks.ts) — task model.
- [server/db/comments.ts](../../../server/db/comments.ts) — already
  supports @mentions; good foundation for PR comment mirroring.
- [server/db/reviews.ts](../../../server/db/reviews.ts) — aligns with
  PR review status.
- [server/db/worktrees.ts](../../../server/db/worktrees.ts) — worktrees
  already know a branch name; natural bridge to PR lookup.
- [server/db/webhooks.ts](../../../server/db/webhooks.ts) — outbound
  webhooks; here we want inbound GitHub webhook handling plus outbound
  API calls, so this is complementary, not redundant.

Reuse:
- `worktrees.branch_name` to locate the PR.
- `comments` table for PR comment mirror.
- R11.3 ingestion auth pattern for GitHub webhook endpoint.

## 3. Data model changes

Migration in `schema.ts`:

- New table `git_host_links` (id TEXT PK, scope_type
  ENUM('project','task'), scope_id TEXT, host ENUM('github','gitlab'),
  remote_id TEXT, remote_url TEXT, remote_kind
  ENUM('issue','pr','mr'), last_synced_at TEXT, sync_direction
  ENUM('pull','push','both')).
- New table `git_host_configs` (id TEXT PK, host, base_url, owner, repo,
  token_enc TEXT, webhook_secret_enc TEXT, active INT).
- Columns on `tasks`: none added — linkage lives in
  `git_host_links`.

## 4. Server surface

New module `server/integrations/git-host/`:
- `github.ts` — thin REST client (no Octokit framework unless already a
  dep). Methods: `getIssue`, `createIssue`, `updateIssue`, `listPRs`,
  `getPR`, `postComment`.
- `gitlab.ts` — mirror.
- `sync.ts` — reconciler:
  - Inbound: `applyRemoteIssue(link, issue)` maps fields
    (title/body/state/labels → task title/description/status/tags).
  - Outbound: `pushTaskToRemote(task_id)` mirrors changes upstream.
  - Conflict policy: last-write-wins with a log entry.
- `webhook.ts` — Express route handling GitHub/GitLab webhooks with
  HMAC signature verification.

REST in `routes.ts`:
- `POST /api/git/configs` (authed), GET/DELETE.
- `POST /api/git/link` — link a task to an issue/PR.
- `POST /api/git/sync/:task_id` — force outbound sync.
- `POST /api/git/webhook/:host` — inbound webhook (verify signature).
- `GET /api/git/prs?task_id=` — list PRs for a task (via worktree branch).

MCP tools:
- `link_task_to_issue`, `sync_task`, `list_task_prs`.

WebSocket: `git_link_created`, `git_sync_completed`, `git_pr_status_changed`.

## 5. Client surface

- Per-task drawer: new "GitHub" section showing linked issue + PRs with
  status chips (draft / open / reviewed / merged), linkable.
- `src/components/GitHostSettings.tsx` — config add/remove + webhook
  URL to paste into the remote.
- FleetView (R11.1): project cards show open PR count.
- TaskBoard: task cards show tiny PR-status dot when linked.

## 6. Test plan

- New `tests/r12-git-sync.test.ts`:
  - Webhook HMAC verification: good signature accepted, bad rejected.
  - Inbound mapping: GitHub issue JSON → task shape, idempotent on
    replay.
  - Outbound: task edit → expected API call (mock `fetch`).
  - Worktree → PR discovery by branch.
- Extend `tests/routes.test.ts` and `tests/mcp-tools.test.ts`.

## 7. Verification

- `npm test`.
- `npm run build`.
- Manual: point a real GitHub repo webhook at a dev tunnel, create an
  issue, confirm a task appears; close the task, confirm the issue
  closes. Ensure two-way flow doesn't loop (sync log prevents echo).

## 8. Out-of-scope

- Bitbucket, Gitea, Azure DevOps — future.
- Full markdown round-trip fidelity (GitHub-flavored → our task body) —
  good-enough plain text for v1.
- PR creation from tasks — read-only + linking only.

## 9. Security

- Tokens encrypted at rest (share helper with R12.1).
- Webhook secret required; reject unsigned.
- Rate-limit inbound endpoint.

## 10. Deprecation companion

None directly. But note: GitHub issue sync may render parts of
`templates.ts` / project seeding less compelling. Re-visit in the next
program review.
