# R12.3 — Information-density pass

**Parent strategy:** [../../PROGRAM-REVIEW-2026-04.md](../../PROGRAM-REVIEW-2026-04.md)

**Task ID:** `609c46d0-e291-454b-a7a9-6f01dba78a99`

## 1. Goal

Make the dashboard **denser**, not prettier. More signal per pixel.
Sparklines in list rows, inline cost/health badges, keyboard-driven
navigation, a real command palette. Realizes the *"intelligence-first,
information-dense, always useful"* stance.

## 3. Scope

This task is **frontend-only**. No schema changes. Reuses data that
FleetView (R11.1), Cost v2 (R11.2), and NL query (R12.1) already
expose.

### Changes

1. **Sparklines everywhere**
   - Each task row: 14-day activity sparkline (activity count) +
     current cost micro-bar.
   - Each agent row: sessions-per-day sparkline.
   - Each project card in FleetView: health sparkline (open blockers
     over time).
   - Build a single inline SVG sparkline component
     `src/components/atoms/Sparkline.tsx` — ~40 LOC, no dep.

2. **Inline badges**
   - Cost chip on task cards when > configurable threshold.
   - Health chip (on-track / at-risk / behind) inline with milestone
     names.
   - Blocker count badge on project + milestone names.

3. **Command palette**
   - `src/components/CommandPalette.tsx` — `⌘K` opens. Fuzzy-matches:
     tasks, projects, milestones, agents, MCP tools, saved queries.
     Reuses `search_tasks` REST endpoint.
   - Actions: jump-to, create-task, trigger digest (R12.1),
     ask-vibe-dash (R12.1), run anomaly scan (R11.2).
   - Kbar-style but without the dep unless already present.

4. **Keyboard navigation**
   - Global: `j/k` row nav, `enter` drill, `g p` go projects, `g f` go
     fleet, `/` focus search, `?` show shortcuts.
   - Implement via one `useKeybindings` hook registered at app root.
   - Document in a `src/components/ShortcutsOverlay.tsx` modal.

5. **Row density mode**
   - Toggle in settings: comfortable / compact / ultra-compact. Persist
     via `saved_filters`-like prefs table (or localStorage; no need for
     schema).

## 4. Non-changes (deferred)
