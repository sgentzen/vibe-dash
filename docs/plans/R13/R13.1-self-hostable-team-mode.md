# R13.1 — Self-Hostable Team Mode

**Goal:** Make it straightforward for a team to run a single shared Vibe Dash instance so all AI agents report to the same dashboard, with documented access control options.

**Approach:** Documentation + deployment artifacts. No SaaS, no centralized auth — teams self-host on their own infrastructure and choose their own access control strategy (VPN, basic auth, Cloudflare Access, IP allowlist).

## Design decisions

**tsx at runtime, not compiled JS:** The `npm start` script already uses `tsx server/index.ts`. The Dockerfile mirrors this. Switching to compiled JS would require separating tsc output paths from vite's `dist/` output, adding complexity for minimal benefit in a developer tool.

**Access control at the proxy, not the app:** Built-in user management is deferred to a future milestone. Teams that need it today have well-understood options (VPN is the recommended path; basic auth for simpler setups). The self-hosting guide covers all practical options.

**Single SQLite file = single process:** The guide explicitly warns against running multiple containers pointing at the same SQLite file. Teams wanting high availability should wait for a future release with proper auth and potential PostgreSQL support.
