# R13.1 — Self-hostable team mode

**Parent strategy:** [../../PROGRAM-REVIEW-2026-04.md](../../PROGRAM-REVIEW-2026-04.md)
**Tier:** C
**Task ID:** `faebc5d7-9afe-4a1d-967c-50642dbb22b4`

## 1. Goal

Ship a documented, open-source path to running Vibe Dash as a
**shared server** for small teams — without any SaaS offering. Uses the
R9b access control already landed. Release valve: teams who want one
Vibe Dash instead of N local ones.

## 2. Context & prior art

Read first:
- [server/index.ts](../../../server/index.ts) — single-user Express
  entry.
- [server/db/schema.ts](../../../server/db/schema.ts) — R9b migration
  added auth tables.
- Any auth module added in R9b (users, sessions, tokens).
- [server/websocket.ts](../../../server/websocket.ts) — WS auth landed
  in R9b; confirm it covers team-mode ingress.

Reuse:
- R9b's auth, users, roles, permissions. Do not re-implement.
- SQLite as the DB — this task does **not** introduce Postgres. Team
  mode = same SQLite, served over HTTP, with auth enforced.

## 3. Architecture

Single self-hostable container:
- Express server (existing) binds to all interfaces when
  `VIBE_DASH_MODE=team`.
- Auth required on every route (reject unauth'd in team mode).
- Multi-user sessions, RBAC, per-project ACLs from R9b.
- SQLite file on a persistent volume; optional Litestream side-car for
  backup (documented, not required).
- Reverse-proxy-friendly (trust `X-Forwarded-*` when
  `VIBE_DASH_TRUST_PROXY=1`).

## 4. Deliverables

1. **Dockerfile + docker-compose.yml** at repo root. Two stages: build
   + runtime. Default `CMD` respects `VIBE_DASH_MODE`.
2. **Systemd unit file** under `deploy/systemd/vibe-dash.service` for
   bare-metal self-hosters.
3. **Deployment docs** `docs/DEPLOYMENT.md`:
   - Docker quick-start.
   - Reverse-proxy snippets (Caddy, nginx).
   - TLS notes (terminate at proxy).
   - Backup/restore guide (Litestream optional).
   - User/role management via CLI.
4. **Team-mode CLI** in [cli/index.ts](../../../cli/index.ts) — add
   subcommands:
   - `vibe-dash user create --email --role`.
   - `vibe-dash user list`.
   - `vibe-dash user reset-password`.
   - `vibe-dash token issue --user`.
5. **Config layer** in `server/config.ts` (new):
   - `MODE=single|team`, `BIND_ADDR`, `TRUST_PROXY`, `MAX_BODY_SIZE`,
     etc. Single source of truth.
6. **Data-at-rest backup hook**: a scheduled `sqlite backup` dump to a
   configurable path; documented, not mandatory.

## 5. Server changes

- `server/index.ts`: respect `MODE`. In team mode require auth
  middleware (from R9b) on *all* routes except health check.
- Rate limits: scale per-user, not global, in team mode.
- Audit log: a tiny append-only `audit_log` table (actor_user_id,
  action, entity_type, entity_id, at) — team mode only. Migration in
  `schema.ts`.
- CSRF protection on cookie-based sessions; documented token-auth path
  for CLI/API usage.

## 6. Client changes

- Login screen exists from R9b; confirm it respects team mode.
- Settings → Users tab (admin only): list/invite/deactivate.
- Display current-user identity in the top bar.

## 7. Test plan

- New `tests/r13-team-mode.test.ts`:
  - In `MODE=team`, unauth'd requests rejected.
  - RBAC: contributor cannot delete project; admin can.
  - Audit log entries written for mutation endpoints.
  - CLI: `user create` end-to-end against a test DB.
- Smoke test `tests/r13-docker.test.ts` (optional; gated on
  `CI=true`): build the Docker image, curl `/api/health`, shut down.

## 8. Verification

- `npm test`.
- `npm run build`.
- `docker compose up` → open http://localhost:3001, login with seeded
  admin, create a second user, revoke, confirm token invalidation.
- Reverse-proxy dry-run with the Caddy snippet from docs.

## 9. Out-of-scope

- SaaS offering / billing / tenant isolation.
- SSO/SAML — document as a future plugin via R10 plugin system.
- Postgres migration (stays SQLite).
- Horizontal scaling.

## 10. Open-source hygiene

- Dockerfile pins versions and uses a non-root user.
- No telemetry/phone-home code.
- Licensing unchanged (Apache 2.0).
- Publish a sample Compose file to
  `examples/compose/team-mode/`.
