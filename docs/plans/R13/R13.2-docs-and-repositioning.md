# R13.2 — Docs + hero examples + homepage reposition

**Parent strategy:** [../../PROGRAM-REVIEW-2026-04.md](../../PROGRAM-REVIEW-2026-04.md)
**Tier:** C
**Task ID:** `4205d780-a6e3-4b32-a346-fa68272dfee2`

## 1. Goal

Make Vibe Dash discoverable, onboardable, and credible. Lead with the
sharpened positioning — *"the open-source project layer for AI-driven
development"* — and ship 60-second integration guides for every AI tool
Vibe Dash ingests from (post-R11.3).

## 2. Context & prior art

Read first:
- [README.md](../../../README.md) — current positioning, feature list,
  quick-start.
- [docs/MCP-SETUP.md](../../../docs/MCP-SETUP.md) — existing setup guide.
- R11.3 integrations (`integrations/*`) — this task ships the docs that
  point at them.
- R13.1 deployment docs — link, don't duplicate.
- [../../PROGRAM-REVIEW-2026-04.md](../../PROGRAM-REVIEW-2026-04.md) —
  positioning source-of-truth.

## 3. Deliverables

1. **Rewritten README.md top section** — lead with:
   > **Vibe Dash** is the open-source project layer for AI-driven
   > development. See the status of every project your AI agents are
   > working on — in one place, on your machine, across any AI tool.
   Hero screenshot: FleetView (R11.1). Below the fold: three 60-second
   integration snippets (Claude Code, Cursor, generic webhook).

2. **One-page integration guides** under
   `docs/integrations/<tool>.md`:
   - `claude-code.md` — MCP config + hook config (covers both R9-era
     explicit MCP and R11.3 hook ingestion).
   - `cursor.md`
   - `codex.md`
   - `copilot-workspace.md`
   - `aider.md`
   - `generic-webhook.md` (for Zed, Windsurf, Roo Code, custom).
   Each: prerequisites, paste-in config, first-success verification
   (`curl /api/health && open http://localhost:3001/fleet`).

3. **Concept docs** under `docs/concepts/`:
   - `data-model.md` — the core entities (project/milestone/task/agent/
     cost/activity/blocker).
   - `fleet-view.md` — how multi-project rollup works.
   - `cost-intelligence.md` — what v2 measures and doesn't.
   - `ingestion.md` — generic ingest schema (referenced from R11.3).

4. **Homepage / GitHub repo polish**:
   - Update GitHub repo `About` blurb to match the new one-liner.
   - Topics: `ai-agents`, `mcp`, `project-management`, `local-first`,
     `vibe-coding`, `observability`, `dashboard`.
   - Add OG image (asset under `assets/og.png`) and link from README.
   - `CONTRIBUTING.md` light refresh; point at R11.4 deprecation doc
     so contributors know what *not* to extend.

5. **Comparison table** — `docs/concepts/compare.md`:
   Vibe Dash vs Braintrust/LangSmith (trace tools), vs Vibe Kanban, vs
   hook-only viewers. Honest, not marketing-y; links strategy doc.

6. **Launch checklist** — `docs/LAUNCH-R13.md`:
   - PR the rewrite.
   - Tag a release.
   - Post to HN, r/ClaudeAI, lobste.rs (content drafts included).
   - Monitor issues; triage playbook.

## 4. Non-changes

- No new product features.
- No schema changes.
- No marketing site. README + docs are it.

## 5. Verification

- Every integration doc: a fresh clone of the repo + following the
  doc = working end-to-end ingestion within 60 seconds (time it).
- All links resolve (`lychee` or `markdown-link-check` in CI; add if
  not present).
- Screenshots reflect the current UI after R11/R12 land (don't ship
  stale pixels).

## 6. Out-of-scope

- Video content.
- Per-platform paid ads.
- Docs site generator (stay with plain markdown in-repo unless
  clearly warranted).

## 7. Sequencing

Do this **last**, after R11 and R12 are stable. Docs that describe
features still being redesigned waste effort.
