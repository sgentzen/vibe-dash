# R11.2 — Cost intelligence v2

**Parent strategy:** [../../PROGRAM-REVIEW-2026-04.md](../../PROGRAM-REVIEW-2026-04.md)
**Tier:** A
**Task ID:** `af0168ec-180c-4c4d-a3b3-3c2f7e21a56e`

## 1. Goal

Turn raw cost data into intelligence that answers *"am I spending my money
well?"* at a glance — per-task ROI, per-milestone totals, model/agent
price-performance scorecards, budget alerts, anomaly detection, and
cross-project leaderboards.

## 2. Context & prior art

Read first:
- [server/db/costs.ts](../../../server/db/costs.ts) — existing
  `logCost`, `getCostSummary`, `getCostTimeseries`, `getCostByModel`,
  `getCostByAgent`.
- [server/db/metrics.ts](../../../server/db/metrics.ts) —
  `completion_metrics` table (lines/tests/duration). **Join cost × metrics
  to compute ROI.**
- [server/db/tasks.ts](../../../server/db/tasks.ts) — task_type field;
  used for type-median anomaly baselines.
- [src/components/DashboardView.tsx](../../../src/components/DashboardView.tsx) —
  current cost charts, which will be superseded (see §9).
- [src/components/AgentDashboard.tsx](../../../src/components/AgentDashboard.tsx) —
  per-agent cost display.

Reuse:
- `costs` table is already well-shaped (agent_id, task_id, milestone_id,
  project_id, model, provider, tokens, cost_usd). No need to widen it for
  basic v2.
- Existing aggregation helpers — extend, don't duplicate.

## 3. Data model changes

Migration in [server/db/schema.ts](../../../server/db/schema.ts):

- New table `budgets` (id TEXT PK, scope_type ENUM('project','milestone'),
  scope_id TEXT, amount_usd REAL, period ENUM('total','monthly'),
  alert_at_pct INT DEFAULT 80, created_at TEXT).
- New table `cost_alerts` (id TEXT PK, budget_id FK, triggered_at TEXT,
  actual_usd REAL, threshold_pct INT, acknowledged_at TEXT NULL).
- New view / cached table `task_cost_roi` (task_id PK, total_cost REAL,
  lines_per_dollar REAL, tests_per_dollar REAL, duration_seconds INT,
  status TEXT) — materialized on task completion.
- Optional: index `idx_costs_task_model` on (task_id, model) for fast
  per-model rollups.

## 4. Server surface

Extend [server/db/costs.ts](../../../server/db/costs.ts):
- `getTaskROI(db, task_id)` — cost × completion_metrics join.
- `getMilestoneROI(db, milestone_id)` — aggregated.
- `getModelScorecard(db, { project_id?, since? })` — for each model: total
  cost, completed tasks, median $/task, median $/line, median $/test,
  avg duration. Sortable.
- `getAgentScorecard(db, { project_id?, since? })` — same shape, for
  agents.
- `getCostLeaderboard(db)` — cross-project: top N projects by 7d/30d
  spend, with % change vs prior window.
- `detectCostAnomalies(db, { project_id, z_threshold=2 })` — flags tasks
  whose cost exceeds N× the median for their `task_type`.

New module `server/db/budgets.ts`:
- `createBudget`, `listBudgets`, `updateBudget`, `deleteBudget`.
- `evaluateBudgets(db)` — called on every `logCost`; inserts
  `cost_alerts` when threshold crossed, broadcasts.

REST in [server/routes.ts](../../../server/routes.ts):
- `GET /api/cost/roi/task/:id`, `GET /api/cost/roi/milestone/:id`.
- `GET /api/cost/scorecard/model`, `GET /api/cost/scorecard/agent`.
- `GET /api/cost/leaderboard`.
- `GET /api/cost/anomalies`.
- CRUD `/api/budgets`, `GET /api/cost/alerts`,
  `POST /api/cost/alerts/:id/ack`.

MCP tools in [server/mcp/tools.ts](../../../server/mcp/tools.ts):
- `get_task_roi`, `get_model_scorecard`, `get_agent_scorecard`,
  `get_cost_anomalies`, `create_budget`, `list_budgets`.

WebSocket: `budget_alert`, `cost_anomaly_detected`.

## 5. Client surface

- New `src/components/CostIntelligenceView.tsx` — replaces the current
  cost subsection of DashboardView. Sections:
  - Budgets & alerts (top, compact strip).
  - Leaderboard (cross-project spend).
  - Model scorecard table (sortable).
  - Agent scorecard table (sortable).
  - Anomalies list (with drill-through to task).
  - ROI sparkline per active milestone.
- `src/components/BudgetEditor.tsx` — inline modal for create/edit.
- Store: extend `store.tsx` with `cost` slice (budgets, alerts,
  scorecards).

Hook into FleetView (R11.1) by surfacing *"$X this week, Y% vs last"* on
each fleet card via `getCostLeaderboard`.

## 6. Test plan

- New `tests/r11-cost-v2.test.ts`:
  - ROI join: given seeded costs + completion_metrics, `getTaskROI`
    returns expected $/line and $/test.
  - Scorecard: deterministic ordering and median math.
  - Anomaly detection: synthetic costs 4× median flagged; 2× not
    flagged (at z=2).
  - Budget evaluation: crossing 80% fires exactly one alert; does not
    re-fire until ack+reset or new period.
- Extend `tests/costs.test.ts`.
- Extend `tests/mcp-tools.test.ts`.

## 7. Verification

- `npm test`.
- `npm run build`.
- Manual: seed 3 agents with varied cost/quality; confirm scorecards
  rank them correctly; set a $1 budget on a milestone, log costs until
  alert fires, verify WebSocket broadcast and UI alert chip.

## 8. Out-of-scope

- Gateway integration (Helicone, OpenRouter) — users still supply
  `log_cost` themselves; ingestion improvements land in R11.3.
- Forecasting / predictive budgets — future, not R11.

## 9. Deprecation companion

DashboardView's current cost charts (timeseries, by-model, by-agent) are
subsumed by CostIntelligenceView. Keep the primitive helpers
(`getCostTimeseries`, etc.) since they're building blocks; remove the
three standalone chart sections from DashboardView once
CostIntelligenceView ships. Note in R11.4 as a deprecation candidate:
`get_cost_timeseries` MCP tool may be redundant with
`get_model_scorecard` + `get_agent_scorecard`.
